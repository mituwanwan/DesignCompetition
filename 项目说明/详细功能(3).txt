# 智慧养老院管理系统 - 必须完成功能清单


---

 负责人1：通用基础模块（第1-3天）

### 1. 用户登录与权限控制系统

#### 功能描述
- 支持护工、管理员、子女三类角色登录
- 账号由管理员统一创建，不开放自主注册
- 登录后自动跳转对应角色界面
- 非对应角色无法访问其他端功能

#### 实现效果
- 登录界面简洁美观，有角色选择
- 密码错误3次锁定账号10分钟（简化实现：使用内存记录失败次数）
- 直接输入其他端URL自动跳回登录页或显示"无权限"提示
- 右上角显示当前登录用户角色和姓名
- 安全退出功能

#### 技术要点
- 密码使用`hashlib.sha256`加盐加密存储
- Flask-Session管理登录状态
- 全局装饰器实现权限拦截

### 2. 老人基础信息管理

#### 功能描述
- 老人信息的增删改查
- 支持按姓名、房间号模糊查询
- 信息导出Excel功能

#### 实现效果
- 列表页展示老人基本信息卡片
- 高级搜索：姓名、房间号、入住状态
- 新增/编辑表单：姓名、性别、年龄、房间号、床位号、紧急联系人、基础病史
- 详情页展示老人完整信息
- 导出Excel按钮，下载包含所有老人信息的表格
- 删除前有确认提示

#### 技术要点
- 使用SQLite数据库存储
- `pandas.to_excel`实现导出功能
- 后端参数校验（必填项、格式验证）

### 3. 全局消息通知系统

#### 功能描述
- 跨端消息同步（简化为轮询机制）
- 报警、留言、任务更新消息通知

#### 实现效果
- 右上角铃铛图标显示未读消息数量
- 点击铃铛展开消息列表
- 新消息有红点提示

#### 技术要点
- AJAX定时请求检查新消息

### 4. 三类角色首页数据展示

#### 功能描述
- 管理员首页：今日入住人数、异常报警数、护工在岗人数统计卡片
- 护工首页：今日待护理老人列表、未处理报警提醒列表
- 子女首页：绑定老人最新健康状态、最近1条消息通知卡片

#### 实现效果
- 管理员首页：三个统计卡片，直观展示关键数据
- 护工首页：待护理老人列表（姓名、房间号、待办事项），未处理报警列表
- 子女首页：绑定老人卡片（头像、姓名、健康状态标签），最近消息展示
- 数据1秒内加载完成，无空值异常
- 响应式设计，适配不同屏幕

#### 技术要点
- SQL聚合查询统计数据
- 缓存机制提升加载速度

负责人2：护工端核心功能（第4-7天）

### 1. 日常护理记录录入

#### 功能描述
- 每日录入老人健康、生活数据
- 字段包含：健康数据（体温、血压、心率、血糖）、饮食、睡眠、情绪

#### 实现效果
- 选择老人后显示录入表单
- 健康数据：体温、收缩压、舒张压、心率、血糖（数字输入，带单位提示）
- 饮食：早/中/晚三餐状态（下拉选择：正常/偏少/偏多/未进食）、饮水量（数字输入）
- 睡眠：前一日睡眠时长（数字输入）、睡眠质量（下拉选择：好/一般/差）
- 情绪：状态（下拉选择：开心/平静/低落/烦躁）、文字备注（选填，文本域）
- 提交成功提示，数据自动保存
- 历史记录查看功能

#### 技术要点
- 完整的后端参数校验（数值范围、必填项）
- 表单分步设计，提升用户体验
- 数据自动同步到管理端、子女端（数据库共享）

### 2. 护理任务管理

#### 功能描述
- 查看管理员分配的护理任务
- 确认完成任务

#### 实现效果
- 任务列表：任务内容、要求完成时间、老人姓名、房间号、状态（待处理/已完成）
- 状态标签颜色区分：待处理（橙色）、已完成（绿色）
- 待处理任务有"确认完成"按钮
- 点击"确认完成"弹出确认框，确认后自动记录完成时间、护工姓名
- 已完成任务不可修改，显示完成时间和完成人
- 任务筛选：全部/待处理/已完成
- 按时间排序功能

#### 技术要点
- 任务状态机管理（待处理→已完成，不可逆向）
- 自动记录操作日志（完成时间、完成人）
- 权限控制（只能处理分配给自己的任务）

### 2. 异常报警中心

#### 功能描述
- 自动+手动报警统一管理
- 报警状态：未处理/处理中/已处理
- 处理后需填写处理结果
- 邮件通知功能（简化实现）

#### 实现效果
- 报警列表：报警时间、老人姓名、房间号、报警类型、状态、处理人、响应时长
- 状态标签：未处理（红色闪烁）、处理中（黄色）、已处理（绿色）
- 自动报警规则配置页面：血压>140/90、心率>100或<60、连续3天睡眠<5小时
- 报警详情页：报警信息、老人基本信息、处理记录、相关图片
- 状态更新：未处理→处理中→已处理
- 处理结果填写（文本域）
- 响应时长自动计算（从报警触发到点击"处理中"的时间差）
- 邮件通知功能（简化为测试模式，显示邮件内容但不实际发送）

#### 技术要点
- 规则引擎实现（基于阈值的自动报警）
- 状态流转控制
- 时间计算与展示
- 邮件模板设计

### 4. 数据统计与可视化

#### 功能描述
- 全院老人健康统计
- 单老人健康趋势图
- 护工工作量统计

#### 实现效果
**全院健康统计**：
- 饼图：高血压、糖尿病、心脏病、其他基础病人数占比
- 柱状图：各楼层健康状况分布
- 数字卡片：在院老人总数、健康老人数、需关注老人数

**单老人健康趋势**：
- 折线图：近7天血压变化（收缩压/舒张压两条线）
- 折线图：近7天心率变化
- 数据表格：详细每日数据
- 老人选择器

**护工工作量统计**：
- 柱状图：各护工月完成护理任务数量
- 表格：护工姓名、完成任务数、护理记录数、上报异常数
- 月份筛选

**通用功能**：
- 图表可放大查看
- 图表导出图片功能
- 响应式布局，适配不同屏幕

#### 技术要点
- Plotly Express基础图表实现
- 数据聚合与计算
- 图表组件复用（管理端、子女端共用）
